Lunaris — Data Viz Site

An interactive data visualisation site built from scratch with vanilla JavaScript and inline SVG, displaying astronomy data through animated, filterable charts.

Year

2024

Role

Front-end Developer

Tools

HTML, CSS, JavaScript, SVG

Overview

Lunaris is a data visualisation site that displays information about planets and moons in the solar system — things like diameter, distance from the Sun, and orbital period. The data is all static (stored in a JS object), but the interface makes it feel interactive and explorable.

The main technical challenge was building the charts without any library like D3 or Chart.js — everything is drawn using inline SVG that gets generated by JavaScript at runtime.

Process

01

Planning the Data Structure

Decided to store all data as a plain JavaScript array of objects. Each object has the same shape (name, type, diameter, distance, orbital period) so the rendering code can be completely generic.

02

Building SVG Charts with JS

This was the hardest part. I had to learn how SVG coordinate systems work — particularly viewBox and how to scale data values to pixel coordinates. The bar chart took several attempts to get right.

03

Filtering and Interactivity

Added a filter bar that lets you switch between planets and moons, and sort by different properties. When the filter changes, the chart re-renders with a short CSS transition on the bars.

04

Responsive Layout

SVG charts are naturally scalable, but I still had to set up the layout with CSS Grid so the charts and the info panel sit side by side on desktop and stack on mobile.

Outcome

The site works fully in the browser with no dependencies — just three files (HTML, CSS, JS). It was the first time I had generated SVG elements dynamically with JavaScript, and it gave me a much better understanding of how charting libraries work under the hood.

If I were to continue this project, I'd pull the data from a real API (like NASA's Open API) instead of hardcoding it, and add a tooltip on hover for each bar that shows the exact values. Both feel doable now that the rendering pipeline is in place.
