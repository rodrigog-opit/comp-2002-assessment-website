Cartly — E-commerce Frontend

A fully responsive e-commerce storefront built with plain HTML, CSS, and JavaScript, featuring a filterable product grid and an animated slide-out cart.

Year

2025

Role

Front-end Developer

Tools

HTML, CSS, JavaScript

Overview

Cartly is a front-end only e-commerce storefront. All the product data lives in a JavaScript array and the page renders the product cards dynamically when it loads. There is no backend — adding to the cart updates a state object in memory and the cart UI reflects that.

I built this mainly to practise working with the DOM at a larger scale than I had before. Previous projects had relatively few moving parts; this one needed filters, a cart with quantity controls, a running total, and a checkout flow — all in vanilla JS.

Process

01

Data & Rendering

Defined a product array with 12 items, each with a name, price, category, and image. A renderProducts() function reads the current filter state and rebuilds the grid — no page reload needed.

02

Filtering

Category filter buttons update a currentFilter variable and call renderProducts() again. The active button gets a highlighted style. Took a while to figure out how to keep the cart state intact when re-rendering the grid.

03

Cart Drawer

The cart is a fixed panel that slides in from the right using a CSS transform transition. It's always in the DOM but off-screen; clicking the cart icon toggles a class that moves it into view.

04

Checkout Flow

A simple three-step flow (cart review → address form → confirmation) built with three hidden divs that get shown one at a time. The address form does basic validation before allowing the next step.

Outcome

Cartly was the most complex front-end project I had built at the time. Managing state across multiple parts of the UI (the product grid, the cart icon badge, the cart drawer) without a framework was genuinely challenging and taught me a lot about how to structure JavaScript in larger projects.

The main thing I'd do differently is separate the data, the state management, and the rendering into clearer functions from the start. Halfway through the project the JS file was getting hard to navigate — I ended up refactoring the cart logic into its own section, which helped a lot.
